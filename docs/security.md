# Security

`wechat-observatory` 需要 Root、LSPosed 和微信进程 Hook。请只在你拥有或被授权管理的设备上使用。

## 主要风险

- Hook 微信进程可能违反微信或相关服务的使用条款。
- 自动化发送和异常行为可能带来账号风控风险。
- 管理台可查看联系人、聊天消息和媒体附件，必须严格限制访问。
- API Key 等同于手机模块凭据，泄露后可上报消息、拉取出站任务。
- 媒体附件可能包含隐私数据，必须有明确的存储和清理策略。

## 管理台保护

生产环境请务必：

- 修改 `BRIDGE_ADMIN_PASSWORD`。
- 把管理台放在内网、VPN 或反向代理鉴权后面。
- 使用 HTTPS。
- 不要在日志、截图、Issue 中暴露密码、API Key、wxid、手机号、聊天内容。

## API Key 生命周期

- 新模块必须使用 Web 管理台生成的 API Key。
- 停用 API Key 后，模块请求会被拒绝，但记录仍保留。
- 删除 API Key 后，对应模块身份会注销。
- 删除后如果手机继续使用旧 Key，不应显示为已注册。
- 泄露后优先停用；确认不再需要时删除。

## 数据存储

MySQL 中会保存：

- API Key 元数据。
- 设备状态。
- 当前微信账号绑定。
- 联系人快照。
- 消息记录。
- 出站队列状态。
- 模块运行状态。

`BRIDGE_MEDIA_DIR` 中会保存图片、语音、视频、文件等附件。公开部署时建议给媒体目录单独做访问控制、备份和清理策略。

## 开源前检查

发布前至少运行：

```bash
rg -n "password|secret|token|api_key|wxid_|phone|banishment|root:" .
```

人工确认命中项是否为示例占位值或测试代码。

不要提交：

- `.env`
- `deploy/docker/.env`
- `deploy/k3s/config.yaml`
- `deploy/k3s/secrets.yaml`
- 构建出的 APK/AAB
- 聊天截图、真实日志、真实数据库导出

## 负责任使用

本项目提供的是微信网关能力，不提供规避风控、批量骚扰、垃圾营销或未授权监控能力。请在合法、合规、取得授权的前提下使用。
